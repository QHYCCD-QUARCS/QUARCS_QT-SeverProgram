# 电子极轴镜模拟校准测试说明（已按当前实现更新）

本文记录电子极轴镜自动极轴校准“模拟入口”的当前实现、前后端链路、图像生成规则和验证方法。

## 功能目标

模拟测试用于在没有赤道仪、极轴镜相机或 INDI/SDK 设备连接的情况下，验证电子极轴镜自动极轴校准界面和显示流程。

它不替代真实校准，只提供一个单独入口：

- 前端命令：`StartPoleMasterAlignmentSimulation`
- 停止命令：复用 `StopAutoPolarAlignment`
- 前端界面：复用 `PoleMasterPolarAlignmentPanel`
- 下行协议：复用 `PoleMasterAlignmentState`、`PoleMasterAlignmentGuideData`、`PoleMasterAlignmentFrameData`，并新增 `PoleMasterAlignmentOverlayData`

## 涉及文件

- 前端入口：`QUARCS_stellarium-web-engine/apps/web-frontend/src/components/AutomaticPolarAlignmentCalibration.vue`
- 前端显示：`QUARCS_stellarium-web-engine/apps/web-frontend/src/components/PoleMasterPolarAlignmentPanel.vue`
- 后端命令入口：`QUARCS_QT-SeverProgram/src/mainwindow.cpp`
- 后端声明：`QUARCS_QT-SeverProgram/src/mainwindow.h`
- 主状态机（真实/模拟共用）：`QUARCS_QT-SeverProgram/src/polemasterpolaralignment.h`
- 主状态机实现（含 `simulationMode` 分支）：`QUARCS_QT-SeverProgram/src/polemasterpolaralignment.cpp`
- 模拟图像脚本：`QUARCS_QT-SeverProgram/src/polemaster_simulation/polemaster_simulate_frame.py`

## 前端流程

点击“模拟测试”后前端会：

1. 将当前极轴校准角色切到 `PoleCamera`。
2. 打开电子极轴镜视图。
3. 设置本地运行态：
   - `isPoleMasterSimulationRunning = true`
   - `isPoleMasterCalibrationActive = true`
   - `isCalibrationRunning = true`
4. 清理旧轨迹/校准显示数据。
5. 发送：

```text
Vue_Command:StartPoleMasterAlignmentSimulation
```

模拟运行中再次点击“停止模拟”，或点击普通停止路径，都会发送：

```text
Vue_Command:StopAutoPolarAlignment
```

模拟入口不检查 `canAutoCalibrate`，因此赤道仪未连接、相机未连接时也可启动。

## 后端流程（当前实现）

`MainWindow::onMessageReceived()` 处理 `StartPoleMasterAlignmentSimulation`。

启动时会先停止并释放旧对象：

- `polarAlignment`
- `poleMasterPolarAlignment`
- `poleMasterAlignmentSimulation`（历史对象，当前模拟入口已不再使用其状态机）

然后调用 `initPoleMasterAlignmentSimulation()` 创建：

```cpp
PoleMasterPolarAlignment(..., simulationMode=true, simulationImageRootPath=imageRoot)
```

模拟模式下不依赖真实硬件：

- 不要求 `dpMount`、`dpPoleScope`、`indi_Client`、SDK 相机句柄可用
- 不调用真实相机采集，改为本地脚本生成 FITS + 预览 JPG
- RA/DEC 使用虚拟位姿推进状态

停止时，`StopAutoPolarAlignment` 会停止：

- 普通自动极轴校准
- 电子极轴镜校准（无论真实或模拟）

## 下行协议（与真实流程保持一致）

状态消息：

```text
PoleMasterAlignmentState:<running>:<state>:<message>:<progress>
```

指导数据（包含 `frameId`）：

```text
PoleMasterAlignmentGuideData:<imageW>:<imageH>:<axisX>:<axisY>:<poleX>:<poleY>:<errorPx>:<errorArcsec>:<frameId>:<hint>
```

图像帧（包含 `frameId`）：

```text
PoleMasterAlignmentFrameData:<fileName>:<imageW>:<imageH>:<frameId>
```

叠加层数据：

```text
PoleMasterAlignmentOverlayData:<json>
```

其中 `json` 会包含 `phase`、`fixedStars`、`rotationSamples`、`axisCandidate`、`tracking`、`quality`、`warnings` 等字段，供前端绘制诊断叠加。

## 状态推进

状态沿用 `PoleMasterAlignmentState`：

1. `INITIALIZING`
2. `FIRST_CAPTURE`
3. `MOVING_RA_FIRST`
4. `SECOND_CAPTURE`
5. `MOVING_RA_SECOND`
6. `THIRD_CAPTURE`
7. `CALCULATING_AXIS`
8. `GUIDING_ADJUSTMENT`
9. `COMPLETED`

前三帧与两次 RA 旋转用于轴心标定；进入 `GUIDING_ADJUSTMENT` 后持续输出 guide data 与 overlay，误差逐步收敛。

## 模拟图像规则

- 图像尺寸：`1024 x 768`
- 参考轴心：`(512, 384)`
- 视场：`11°(RA) x 8°(DEC)`
- 像素比例：`4.0 arcsec/pixel`

图像生成逻辑：

- 优先使用 HIP 星表；缺失时可退化。
- 基于极点中心 + roll + 视场投影到像素平面。
- 按曝光映射可见极限星等，仅渲染 `mag <= limitingMagnitude(exposure)` 的星。
- 亮星先饱和，叠加背景噪声。

文件命名：

```text
PoleMasterSim_0000.jpg
PoleMasterSim_0001.jpg
...
```

写入路径：

- 优先写入 `vueDirectoryPath`
- 下发帧消息时尝试将图像在 `vueImagePath` 建链或复制，保障前端静态路径可访问

## 阶段驱动规则

机械轴心固定为参考点，真实天极按状态生成：

- `FIRST_CAPTURE`：初始偏差
- `SECOND_CAPTURE/THIRD_CAPTURE`：随 RA 旋转采样变化
- `GUIDING_ADJUSTMENT`：偏差按收敛轨迹递减并叠加轻微扰动

误差计算：

```text
errorPx = hypot(axisX - poleX, axisY - poleY)
errorArcsec = errorPx * pixelScaleArcsecPerPixel
```

完成条件与真实流程一致：

- `errorArcsec <= 30`
- 连续 `3` 帧满足阈值

## 手动验证

前端验证：

1. 不连接赤道仪和极轴镜相机。
2. 打开“极轴校准”面板。
3. 点击“模拟测试”。
4. 确认界面切到电子极轴镜。
5. 确认显示模拟星图、轴心、真实天极。
6. 确认误差逐步下降，且 `frameId`、overlay 诊断持续更新。
7. 点击“停止模拟”，确认状态停止且不再刷新图像。

## 日志排查

后端日志建议检索：

```text
StartPoleMasterAlignmentSimulation
initPoleMasterAlignmentSimulation
PoleMasterPolarAlignment
PoleMasterAlignmentGuideData
PoleMasterAlignmentFrameData
PoleMasterAlignmentOverlayData
```

常见问题：

- 点击按钮无切换：检查前端是否发送 `StartPoleMasterAlignmentSimulation`
- 后端无响应：检查 `MAIN.log` 是否收到命令
- 星图不显示：检查 `PoleMasterSim_*.jpg` 是否生成、以及 `vueDirectoryPath` 与 `vueImagePath` 链接/复制是否成功
- 误差不变化：检查 `PoleMasterAlignmentGuideData` 的 `poleX/poleY` 与 `frameId` 是否逐帧更新
- 叠加不显示：检查是否收到 `PoleMasterAlignmentOverlayData`

检查最近日志：

```bash
grep -RniE "StartPoleMasterAlignmentSimulation|initPoleMasterAlignmentSimulation|PoleMasterPolarAlignment|PoleMasterAlignmentGuideData|PoleMasterAlignmentFrameData|PoleMasterAlignmentOverlayData" \
  ~/workspace/QUARCS/QUARCS_QT-SeverProgram/src/BUILD/logs/MAIN.log | tail -100
```
