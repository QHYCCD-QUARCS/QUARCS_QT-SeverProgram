# 后端日志读取与崩溃排查指南

本文记录当前通过 SSH 读取树莓派后端日志、定位 Qt Server 崩溃、确认 PoleMaster 图像链路的常用方法。

## 远端信息

- 主机：`192.168.1.103`
- 用户：`quarcs`
- 密码：`quarcs`
- 后端运行目录：`~/workspace/QUARCS/QUARCS_QT-SeverProgram/src/BUILD`
- 后端日志目录：`~/workspace/QUARCS/QUARCS_QT-SeverProgram/src/BUILD/logs`
- 主要可执行文件：`~/workspace/QUARCS/QUARCS_QT-SeverProgram/src/BUILD/client`

## 登录与基础检查

```bash
ssh quarcs@192.168.1.103
```

如果本机安装了 `sshpass`，可用非交互命令读取日志：

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.103'hostname; date; whoami; pwd'
```

查看后端相关进程：

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.103\
'ps -eo pid,ppid,stat,comm,args | grep -Ei "QUARCS|qt|server|pole|stellarium|websocketclient" | grep -v grep || true'
```

## 找日志文件

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.103\
'cd ~/workspace/QUARCS/QUARCS_QT-SeverProgram/src/BUILD && ls -lahtr && find logs -maxdepth 1 -type f -printf "%TY-%Tm-%Td %TH:%TM:%TS %s %p\n" | sort'
```

常用日志：

- `logs/MAIN.log`：主流程、WebSocket 指令、极轴校准状态、系统消息
- `logs/CAMERA.log`：相机曝光、FITS 保存、INDI/SDK 相机回调
- `logs/GUIDER.log`：内置导星、导星曝光、导星预览
- `logs/MOUNT.log`：赤道仪状态与控制
- `logs/FOCUSER.log`：电调连接与移动

## 读取最近日志

一次读取主要日志尾部：

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.107 \
'cd ~/workspace/QUARCS/QUARCS_QT-SeverProgram/src/BUILD/logs && for f in MAIN.log CAMERA.log GUIDER.log FOCUSER.log MOUNT.log; do echo ===$f===; tail -220 "$f" 2>/dev/null; done'
```

按关键字搜索最近异常：

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.107 \
'cd ~/workspace/QUARCS/QUARCS_QT-SeverProgram/src/BUILD/logs && grep -RniE "segfault|segmentation|crash|error|failed|PoleMaster|PolarAlignment|StartAutoPolar|Exposure completed|Image saved" . | tail -300'
```

按某一分钟定位崩溃窗口，例如 `15:19`：

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.107 \
'cd ~/workspace/QUARCS/QUARCS_QT-SeverProgram/src/BUILD/logs && echo MAIN && grep -n "15:19" MAIN.log | tail -100; echo CAMERA && grep -n "15:19" CAMERA.log | tail -100; echo GUIDER && grep -n "15:19" GUIDER.log | tail -100'
```

## 崩溃排查顺序

1. 先看 `MAIN.log` 中最后一条收到的前端命令，例如 `StartAutoPolarAlignment:PoleCamera`。
2. 再看 `CAMERA.log` 是否出现曝光开始、`Image saved to ...`、`Exposure completed...`。
3. 同时看 `GUIDER.log` 是否在同一时间停止导星或还有上一帧曝光未完成。
4. 如果日志同时停止，最后几行通常就是崩溃窗口。
5. 用最后一个函数名或消息反查源码，例如：

```bash
rg -n "savePoleMasterPreviewAsJPG|PoleMasterAlignmentFrameData|startPoleCameraSingleCapture|notifyPolarAlignmentCaptureReady" ~/workspace/QUARCS/QUARCS_QT-SeverProgram/src
```

## 系统级崩溃信息

查看 systemd core dump 列表：

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.107 \
'coredumpctl --no-pager list 2>/dev/null | tail -20 || true'
```

查看内核 segfault 记录：

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.107 \
'dmesg --ctime 2>/dev/null | grep -Ei "segfault|trap|core|client|QUARCS" | tail -120 || true'
```

如果没有 core dump 或 dmesg 记录，就以应用日志最后窗口为主。

## PoleMaster 图像链路检查

PoleMaster 预览图保存位置：

- 源文件：`/dev/shm/PoleMasterImage_*.jpg`
- nginx 静态目录软链接：`/var/www/html/img/PoleMasterImage_*.jpg`
- 前端访问 URL：`http://<设备IP>:8080/img/PoleMasterImage_*.jpg`

检查文件和软链接：

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.107 \
'ls -lah /var/www/html/img/PoleMasterImage_* /dev/shm/PoleMasterImage_* 2>/dev/null | tail -20 || true'
```

检查 nginx 是否能访问：

```bash
sshpass -p quarcs ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null quarcs@192.168.1.107 \
'latest=$(ls -t /var/www/html/img/PoleMasterImage_* 2>/dev/null | head -1); f=$(basename "$latest"); echo file=$f; curl -sS -I --max-time 3 "http://127.0.0.1:8080/img/$f" || true'
```

正常应返回：

```text
HTTP/1.1 200 OK
Content-Type: image/jpeg
```

## 本次典型案例

现象：

- Qt Server `exitCode: 11`
- 日志停在 `Image saved to /dev/shm/polecamera.fits` 和 `Exposure completed`
- 前端不显示 PoleMaster 图像

定位：

- 段错误发生在 FITS 保存完成后的 PoleMaster 预览生成阶段。
- `savePoleMasterPreviewAsJPG()` 中 `preview8` 未分配内存就传给 `Tools::Bit16To8_Stretch()`，后者直接写 `img8.data[s]`，触发空指针写入。
- 前端不显示图像是因为 `PoleMasterAlignmentFrameData` 只包含 `fileName/imageWidth/imageHeight`，但前端接收函数曾要求必须有 `sessionId`，导致消息被丢弃。

相关源码：

- `src/mainwindow.cpp`
  - `savePoleMasterPreviewAsJPG()`
  - `notifyPolarAlignmentCaptureReady()`
  - `startPoleCameraSingleCapture()`
- `src/main.cpp`
  - `qRegisterMetaType<QVector<QPointF>>()`
- 前端：
  - `apps/web-frontend/src/App.vue`
  - `apps/web-frontend/src/components/AutomaticPolarAlignmentCalibration.vue`
  - `apps/web-frontend/src/components/PoleMasterPolarAlignmentPanel.vue`
