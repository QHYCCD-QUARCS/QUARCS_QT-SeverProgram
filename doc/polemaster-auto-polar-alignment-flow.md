# 电子极轴镜自动极轴校准流程

本文档描述当前 `PoleCamera` 电子极轴镜自动校准流程。实现涉及前端 `AutomaticPolarAlignmentCalibration.vue`、`PoleMasterPolarAlignmentPanel.vue`，以及后端 `PoleMasterPolarAlignment` 和 `MainWindow` 的 WebSocket 转发。

## 一、总体目标

当前流程不是原 PoleMaster 人工向导的逐步复刻，而是“全自动测量 + 可视化证据展示”：

- 后端自动拍摄、板解、识别星点、旋转 RA、计算赤经轴机械旋转中心。
- 在轴心拟合并回 Home 后，进入纯快速星点跟踪引导，不再每帧板解。
- 前端展示每一步的证据，包括固定亮星对应结果、旋转采样轨迹、轴心候选和最终误差线。
- 用户不需要双击 Polaris、选择参考星或点击 Success/Correct。
- 用户只在最终阶段手动调节赤道仪高度和方位螺丝，让真实天极靠近机械轴心。

最终目标是让“真实天极”与“赤经轴机械旋转中心”在极轴镜图像中重合。

## 二、WebSocket 数据通道

电子极轴镜流程保留原有三类消息：

```text
PoleMasterAlignmentState:{running}:{state}:{message}:{progress}
PoleMasterAlignmentFrameData:{fileName}:{imageW}:{imageH}:{frameId}
PoleMasterAlignmentGuideData:{imageW}:{imageH}:{axisX}:{axisY}:{poleX}:{poleY}:{errorPx}:{errorArcsec}:{frameId}:{hint}
```

新增一类可视化叠加消息：

```text
PoleMasterAlignmentOverlayData:{json}
```

`json` 至少包含：

- `phase`：当前可视化阶段，例如 `field-confirmation`、`ra-rotation-1`、`axis-fit`、`return-home`、`guiding`。
- `imageW/imageH`：叠加坐标对应图像尺寸。
- `fixedStars`：南/北半球各自固定亮星的预测位置、检测对应位置和匹配状态。
- `templateRoles`：固定模板星语义映射（A/B/D + 其余模板星）。
- `rotationSamples`：三帧旋转采样点和参与展示的星点。
- `axisCandidate`：当前轴心候选、半径和残差。
- `selectedTrackStars`：引导阶段参与快速跟踪的筛选星点。
- `tracking`：跟踪状态（模式、锁定状态、锁定置信度、丢失帧数、参与跟踪星点数）。
- `quality`：检测星点数量、参与跟踪星点数量、筛选策略、固定亮星对应数量、曝光、RA 实际旋转角、拟合残差等。
- `warnings`：降级、星点不足、RA 旋转不足等提示。

旧前端如果忽略 `PoleMasterAlignmentOverlayData`，仍能通过原有 GuideData 显示最终误差。

## 三、自动流程

### 1. 初始化

前端选择 `PoleCamera` 后发送：

```text
StartAutoPolarAlignment:PoleCamera
```

后端检查赤道仪、电子极轴镜、焦距、芯片尺寸和本地坐标。启动时记录起始 RA/DEC，并关闭赤道仪跟踪。

### 2. 星场确认

后端拍摄第一帧并执行近天极板解。板解成功后：

- 用 WCS 投影真实天极。
- 提取图像中的星点。
- 根据解析结果投影当前半球固定亮星，并与检测星点做对应判断。
  - 固定亮星模板固定为 5 颗，优先选择在短曝光下稳定可见的亮星。
  - 暗星不再从模拟星图中屏蔽；会稳定显示，但以更暗亮度、更小星点绘制。
  - 模板语义与人工 PoleMaster 对齐：`B=Polaris/σ Oct`，`A/D=参考亮星`。
- 发送 overlay，让前端绘制固定亮星对应结果。

如果星点数量不足，后端会自动增加曝光并重拍，最多 3 次。重试后仍不足时，流程继续保留板解结果，并在前端显示 warning；后续轴心拟合可能降级。

### 3. 第一次 RA 旋转采样

后端自动将 RA 轴旋转配置角度，当前默认 35°，并等待赤道仪 Idle。到位后拍摄第二帧、板解、提取星点并发送 overlay。

前端显示第二帧星点和旋转采样轨迹。

### 4. 第二次 RA 旋转采样

后端再次沿同方向旋转 RA 轴，默认再转 35°，并拍摄第三帧。后端记录实际 RA 变化量，如果最近一次实际旋转小于 30°，会在 overlay warning 中提示。

### 5. 轴心拟合

后端优先尝试多星旋转中心：

- 使用三帧检测星点进行粗匹配。
- 对可用星点轨迹进行三点圆心估计。
- 汇总多个圆心候选，得到机械轴心、半径和残差。

如果多星匹配不足，自动降级为真实天极三点圆拟合。降级不会直接中断流程，但会在前端显示。

拟合完成后前端显示：

- 参与旋转采样的点。
- 机械轴心候选。
- 拟合半径和残差。
- 使用的方法：`multi-star` 或 `fallback-pole`。

### 6. 返回 Home

轴心拟合完成后，后端尝试返回启动时记录的 RA/DEC。返回阶段会发送 `return-home` overlay。返回失败不会清空已计算轴心，但用户应检查线缆和赤道仪状态。

### 7. 实时手动调整（纯快速跟踪）

后端在进入第7步时先做一次跟踪锚定初始化，然后每帧只做快速跟踪（不再板解）：

- 初始化锚定：使用第6步最后成功板解帧建立锁定星、机械轴心和天极偏移基准。
- 每帧跟踪：提取星点后按“Top-K + 最小间距 + 边缘抑制”筛选参与星点，再用 LK + 邻域匹配更新锁定星位置；并按星点数量窗口自适应调整曝光，抑制过密星点造成的误匹配。
- 天极更新：由锁定星位移和初始偏移推算当前天极位置，进而计算与机械轴心误差。
- 失锁恢复：先局部扩大半径重锁，再全图重选亮星重锁；连续失锁超过阈值则失败并提示重跑校准。

误差计算保持不变：

```text
dx = axisX - poleX
dy = axisY - poleY
errorPx = sqrt(dx^2 + dy^2)
errorArcsec = errorPx * pixelScaleArcsecPerPixel
```

前端绘制绿色机械轴心、红色真实天极和黄色误差线。用户只使用高度和方位螺丝调节赤道仪。误差连续 3 帧小于等于 30 角秒后完成。

## 四、前端显示

`PoleMasterPolarAlignmentPanel.vue` 在同一个画布中叠加：

- 蓝色圆圈：固定亮星已检测并对应。
- 红色圆圈：固定亮星未检测到，在解析预测位置提示。
- 黄色点/轨迹：RA 旋转采样。
- 绿色十字/圆：机械轴心候选和最终轴心。
- 红色圆：真实天极。
- 黄色线：真实天极到机械轴心的误差。

右侧诊断区显示：

- 当前阶段。
- 检测星点数量/参与跟踪星点数量/固定亮星对应数量。
- 最近 RA 实际旋转角/轴心残差。
- 跟踪锁定状态与锁定置信度。
- warning 文本。

步骤条显示：准备、星场确认、固定星确认、旋转采样、轴心拟合、返回/调整、完成。

## 五、模拟流程（同状态机，仅替换图像源）

前端可通过“模拟测试”触发：

```text
StartPoleMasterAlignmentSimulation
```

模拟流程保留独立启动命令，但不再使用独立流程状态机。`StartPoleMasterAlignmentSimulation` 会创建 `PoleMasterPolarAlignment(simulationMode=true)`，沿用与真实流程完全一致的状态推进与引导逻辑，仅替换拍摄与位姿来源。

模拟流程不依赖真实硬件，会生成 `PoleMasterSim_*.jpg`，并发送与真实流程一致的：

- `PoleMasterAlignmentState`
- `PoleMasterAlignmentFrameData`
- `PoleMasterAlignmentGuideData`
- `PoleMasterAlignmentOverlayData`

其中：

- 标定阶段 RA 旋转由同一 `moveRaAxis(config.raRotationAngle)` 驱动（虚拟位姿）。
- 调整阶段极点偏差由同一引导环路驱动并逐步收敛（带轻微扰动）。
- overlay 包含固定亮星对应结果、三帧旋转轨迹、轴心候选和逐步收敛的真实天极。

该模式用于验证前端可视化、协议兼容和无硬件演示，同时避免真实/模拟流程漂移。

### 模拟图像规则

- 视场固定：`11° x 8°`。
- 曝光影响可见极限星等，按曝光-星等表分段插值。
- 仅渲染 `mag <= limitingMagnitude(exposure)` 的恒星，暗于阈值的星直接过滤。
- 星点亮度随曝光提升并有饱和上限，亮星先饱和。
- 自动增曝重拍时，图像星点数与亮度会同步变化。

## 六、与原人工 PoleMaster 流程对照

| 原人工步骤 | 当前自动流程 |
| --- | --- |
| 调曝光让 Polaris 附近看到更多星点 | 自动检测星点数量，必要时提高曝光重拍，并显示星点数量 |
| 双击 Polaris 并旋转模板 | 自动投影固定亮星，前端用蓝/红圆圈显示是否检测对应 |
| 选择一颗非 Polaris 参考星 | 后端优先使用多星旋转中心，不需要人工选星 |
| 手动旋转 RA 轴至少 30° | 后端自动旋转 RA，记录并显示实际旋转角 |
| 再次选择同一颗参考星 | 后端自动匹配三帧星点轨迹 |
| 第二次继续旋转 RA | 后端自动执行第二次旋转采样 |
| 回 Home 并确认圆心 | 后端尝试返回起始 RA/DEC，并显示 `return-home` 阶段 |
| 再次匹配模板 | 保留模板语义叠加；引导阶段改为纯快速跟踪并持续输出质量叠加 |
| 调高度/方位移动北极星 | 当前最终阶段调节真实天极到机械轴心 |
| Start Monitor | 实时调整阶段自动开始 |
| 移动 Pole rectangle 到 Axis circle | 前端显示红色真实天极靠近绿色机械轴心 |

## 七、南半球处理

真实天极投影会根据观测地纬度选择：

- 北半球：`DEC = +89.9999`
- 南半球：`DEC = -89.9999`

北半球和南半球都会使用一组固定亮星做可视化确认。流程仍通过板解、星点检测、多星旋转中心和真实天极投影完成。

固定亮星模板约束：

- 每个半球使用 5 颗模板星。
- 模板星优先标准为：短曝光下可见且稳定可检测。
- 模板语义固定：`B` 为极轴参考星（北半球 Polaris / 南半球 σ Oct），`A/D` 为参考亮星。
- 低亮度星可存在于背景星表，但当曝光不足时会被可见星等阈值过滤，不进入当帧检测统计。

## 八、失败与降级

常见 warning 或失败原因：

- 星点数量不足：自动提高曝光重试，仍不足则显示 warning。
- RA 实际旋转不足 30°：显示 warning，建议检查赤道仪动作。
- 多星匹配不足：自动降级到真实天极三点圆拟合。
- 三点近似共线或半径过小：轴心拟合失败。
- 板解失败、相机采集超时、WCS 工具不可用：流程失败。
- 第7步连续失锁超过阈值：纯跟踪恢复失败，需重新执行校准。

设计原则是：1-6 步优先保证板解与轴心拟合正确性；第7步优先保证实时性与跟踪稳定性。只有采集、板解、轴心拟合或连续失锁恢复失败时才停止校准。
