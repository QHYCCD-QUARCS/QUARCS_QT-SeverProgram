# 2026-04-28 自动极轴校准与极轴引导锁星算法 V2 设计说明

## 1. 背景与问题
自动极轴校准由后端 `src/autopolaralignment.{h,cpp}` 实现，目标是在赤道仪、拍摄相机和本地 plate solve 可用的情况下，自动完成三点测量、计算假极轴与真极轴偏差，并在最后的用户调整阶段持续给出图像内箭头和方位/高度偏差。

当前图像内引导目标：
- 以全图多星匹配作为主决策链路。
- 单星仅用于局部精修，不再作为唯一真值来源。
- 失败降级采用分层恢复，避免单帧失败立即触发重锚。
- 在角秒级精调时，补偿星场持续漂移，并根据测量噪声动态放宽达标阈值，避免要求用户追逐不可测的瞬时误差。

## 2. 自动极轴校准总流程
完整流程由 `PolarAlignmentState` 驱动：

1. `INITIALIZING`
   初始化运行状态、清空历史三点测量、清空图像引导状态和目标缓存。

2. `CHECKING_POLAR_POINT`
   读取赤道仪 JNow RA/DEC，判断是否接近天极。若 DEC 在极区，先移动 DEC 轴脱离极点，避免极点附近 WCS/RA 退化。

3. 三点拍摄与解算
   依次执行 `FIRST_CAPTURE`、`SECOND_CAPTURE`、`THIRD_CAPTURE`，两次 RA 旋转之间拍摄三张图。每个点都会拍照、等待文件、plate solve、读取 `SloveResults`。短曝光失败时进入长曝光重试；仍失败时按当前阶段尝试 DEC/RA 避障移动。

4. `CALCULATING_DEVIATION`
   使用三点严格球面几何法计算假极轴和偏差。核心思路是把三次解算中心点视作天球单位向量 `q1/q2/q3`，它们绕赤道仪实际极轴形成一条小圆；通过三点拟合小圆球心得到假极轴 `c_hat`，再与真实天极 `p` 比较。

5. 固定目标点计算
   当前视场中心点 `S` 通过“把假极轴旋到真极轴”的最小旋转映射到目标点 `T`。`targetRA/targetDEC` 在一次校准流程内保持固定，是后续所有指导调整的天球真值。

6. `GUIDING_ADJUSTMENT`
   循环拍照并发送两类指导：
   - 天球/地平偏差：用当前 fresh solve 的 RA/DEC 与固定目标 `targetRA/targetDEC` 计算方位/高度调整量。
   - 图像内引导：把固定目标投影到当前图像，锁定一颗可见星，显示这颗星应移动到的位置和箭头。

7. 结束
   用户停止时，如果目标已缓存，会用最新解算结果输出当前精度。自动图像引导进入 `done` 时表示后端认为当前残差已低于可测阈值并稳定了若干帧。

## 3. 三点几何计算与固定目标
三点严格几何法的关键是不要直接线性比较 RA/DEC，而是在球面上处理：

1. 解算结果转单位向量：
   `q_i = radec_to_vec(RA_i, DEC_i)`

2. 三点拟合小圆球心：
   理想情况下 `c` 同时垂直于 `(q1-q2)` 和 `(q1-q3)`，所以：
   `c_hat = normalize((q1 - q2) x (q1 - q3))`

3. 选择正确半球：
   小圆球心有正负两个方向，代码根据真实极点方向选择与真极更接近的一侧作为假极轴。

4. 偏差量：
   总偏差是 `angle(c_hat, p)`；方位/高度提示会结合观测者经纬度转到当前地平坐标系。

5. 固定目标：
   对当前点 `S` 应用最小旋转 `R(c_hat -> p)`，得到目标点 `T = R * S`。之后用户调整时，系统始终引导当前解算点向这个固定目标靠近，而不是每帧重新定义天球目标。

## 4. 最新图像内引导流程
`INIT -> GLOBAL_MATCH -> TRACKING_FUSED -> RECOVERING_MULTI -> REANCHOR_SOLVE -> DONE/FAILED`

当前代码中的主要状态：
- `INIT`：建立初始锁星与固定目标。
- `TRACKING_FUSED`：多星全局匹配 + 单星局部精修的正常跟踪态。
- `RECOVERING_MULTI`：局部精修失败后，依赖多星匹配恢复。
- `REANCHOR_SOLVE`：多星恢复失败时触发 fresh solve 重锚。
- `DONE/FAILED`：收敛或失败终态。

## 5. 图像引导状态机定义与转移条件表
| 当前态 | 触发条件 | 下一态 | 说明 |
|---|---|---|---|
| init | 可选星成功 | tracking_fused | 建立初始锁星与固定目标 |
| init | 无可选星 | recovering_multi | 等待多星质量恢复 |
| tracking_fused | 单星精修成功且质量达标 | tracking_fused | 正常闭环 |
| tracking_fused | 单星精修失败 | recovering_multi | 不立即 reanchor |
| recovering_multi | 全图匹配质量恢复 | tracking_fused | 从多星恢复回融合跟踪 |
| recovering_multi | 连续丢失帧超阈值 | reanchor_solve | 触发全局重锚 |
| reanchor_solve | 本帧 fresh solve 成功 | init | 用新解重建目标 |
| reanchor_solve | 超时 | failed | 进入失败态 |
| failed | 冷却后允许重试 | reanchor_solve | 自动恢复入口 |
| tracking_fused | 连续稳定帧达阈值 | done | 达标收敛 |

## 6. 图像引导核心模型
### 6.1 全图模型估计
上一帧点集 `P` 与当前帧点集 `Q` 先进行近邻候选匹配，再进行 RANSAC 估计：

1. `Similarity (estimateAffinePartial2D)`
2. `Affine (estimateAffine2D)`
3. `Translation-only`（均值位移降级）

### 6.2 目标传播
对估计目标点和锁星点使用 2x3 仿射矩阵传播：

`[x', y']^T = A * [x, y, 1]^T`

### 6.3 局部精修
在传播后位置附近做局部跟踪（LK + 最近邻点回归），搜索半径采用自适应策略：

`radius = clamp(base + vel_gain * reproj_err, base, max)`

### 6.4 固定天球目标到锁星像素目标
每次 fresh solve 后，后端会把固定天球目标 `targetRA/targetDEC` 投影到当前图像，得到 `solveTargetPx`。初次锁星时：

`frameDeltaPx = imageCenterPx - solveTargetPx`

`lockStarFixedTargetPx = selectedStarPx + frameDeltaPx`

含义是：如果用户通过机械方位/高度调整让当前图像中心到达固定天球目标，那么被锁定的这颗星也应移动同样的像素偏移。这个目标是“固定天球目标在当前图像坐标中的等效落点”，不是恢复旧的“直接把某颗星当成唯一真值”的逻辑。

### 6.5 fresh solve 平滑重锚
角秒级时，仅靠上一帧仿射传播会积累 WCS/匹配误差。当 `imageGuideFreshSolveThisFrame == true` 且锁星仍有效时，后端会用最新 WCS 重新计算锁星目标：

`desiredTargetPx = currentStarPx + (imageCenterPx - solveTargetPx)`

然后用低通更新：

`lockStarFixedTargetPx = oldTarget * (1 - alpha) + desiredTargetPx * alpha`

默认 `alpha=0.35`。这样目标会跟随 fresh solve 的真实 WCS 缓慢校正，不会因为单次解算噪声而大跳。

### 6.6 漂移补偿
角秒级精调中，星点会因为跟踪关闭、极轴残差、大气折射或机械微动持续漂移。系统从多星全局变换的中心点位移估计整场速度：

`v_px_per_sec = (A * imageCenterPx - imageCenterPx) / dt`

并进行低通滤波。达标判断时不直接使用瞬时 `lockStarFixedTargetPx - currentStarPx`，而是扣除曝光期间可预期的漂移项：

`compensatedDeltaPx = (lockStarFixedTargetPx - currentStarPx) - v_px_per_sec * exposureSec * factor`

其中 `factor` 默认 0.5，近似把曝光中点作为测量时刻。前端仍可显示原始箭头，后端完成判定使用补偿后的残差。

### 6.7 动态精度阈值
原先 `done` 只看固定像素阈值：`totalRemainPx < QUARCS_POLAR_DONE_THRESHOLD_PX`。在角秒级时，如果测量噪声已经大于残差，继续追零会导致轨迹不可预测。

当前动态阈值：

`measurementNoisePx = sqrt(lastSolveReprojErr^2 + matchReprojErr^2 + driftDuringExposure^2)`

`dynamicThresholdPx = max(doneThresholdPx, doneThresholdArcsec / arcsecPerPixel, sigmaFactor * measurementNoisePx)`

达标判断使用：

`compensatedRemainPx < dynamicThresholdPx`

并要求连续 `QUARCS_POLAR_DONE_STABLE_FRAMES` 帧满足。V2 JSON 的 `precision` 字段会输出补偿后残差、动态阈值、噪声估计和漂移速度，便于前端展示或日志排查。

### 6.8 分段目标
当最终目标偏远或近边缘时，计算可视分段目标，保证箭头在当前图内可执行。

## 7. 参数表（变量名/默认值/作用/调参建议）
| 变量名 | 默认值 | 作用 | 调参建议 |
|---|---:|---|---|
| QUARCS_POLAR_MATCH_MAX_STARS | 200 | 星点上限 | 性能不足时下调到 120~160 |
| QUARCS_POLAR_MATCH_MIN_INLIERS | 8 | 最小内点数 | 稀疏星场可降到 6 |
| QUARCS_POLAR_MATCH_MIN_INLIER_RATIO | 0.35 | 最小内点比例 | 抖动大场景可降到 0.30 |

| QUARCS_POLAR_TRACK_BASE_RADIUS_PX | 18 | 基础跟踪半径 | 精度优先可降到 14 |
| QUARCS_POLAR_TRACK_MAX_RADIUS_PX | 80 | 最大跟踪半径 | 防误匹配可降到 60 |
| QUARCS_POLAR_TRACK_RADIUS_VEL_GAIN | 1.2 | 半径增益 | 响应慢时小幅上调 |
| QUARCS_POLAR_RECOVER_MAX_FRAMES | 12 | 恢复态最大帧数 | 重锚过慢可下调到 8~10 |
| QUARCS_POLAR_REANCHOR_TIMEOUT_MS | 25000 | 重锚超时 | 网络慢时上调 |
| QUARCS_POLAR_FULL_SOLVE_MAX_INTERVAL | 8 | 保底 full solve 间隔 | 性能紧张时上调 |
| QUARCS_POLAR_EDGE_MARGIN_RATIO | 0.08 | 边缘保护比例 | 边缘误触发高可下调 |
| QUARCS_POLAR_DONE_THRESHOLD_PX | 3.0 | 像素级基础达标阈值 | 严格达标可下调到 2.5 |
| QUARCS_POLAR_DONE_STABLE_FRAMES | 8 | 达标稳定帧 | 稳定性不足可上调 |
| QUARCS_POLAR_DRIFT_COMPENSATION | 1 | 是否启用漂移补偿 | 角秒级精调建议开启；若漂移估计异常可临时关掉 |
| QUARCS_POLAR_DRIFT_LOWPASS_ALPHA | 0.25 | 漂移速度低通系数 | 漂移变化快可上调，噪声大可下调 |
| QUARCS_POLAR_DRIFT_EXPOSURE_NOISE_FACTOR | 0.5 | 曝光期间漂移补偿/噪声因子 | 默认按曝光中点估计；长曝光可适当上调 |
| QUARCS_POLAR_FRESH_SOLVE_REANCHOR_ALPHA | 0.35 | fresh solve 重锚平滑系数 | 目标跳动大时下调；WCS 修正响应慢时上调 |
| QUARCS_POLAR_DYNAMIC_THRESHOLD_SIGMA | 2.5 | 动态阈值噪声倍数 | 要更严格可下调；现场抖动大可上调 |
| QUARCS_POLAR_DONE_THRESHOLD_ARCSEC | 0.0 | 角秒级最小达标阈值；0 表示不用固定角秒下限 | 若希望 5" 内即认为可接受，可设 5 |

## 8. 协议说明
`PolarImageGuidanceDataV2:<json>`

JSON 字段：
- `frameId`
- `mode` (`init/global_match/tracking_fused/recovering_multi/reanchor_solve/done/failed`)
- `stateVersion` (=2)
- `lock` (`hasLock/confidence/stableFrames/lostFrames`)
- `target` (`currentX/currentY/displayTargetX/displayTargetY/finalTargetX/finalTargetY/inFrame`)
- `match` (`model/inliers/inlierRatio/reprojErrPx`)
- `precision` (`compensatedRemainArcsec/dynamicDoneThresholdArcsec/measurementNoiseArcsec/driftPxPerSec`)
- `solve` (`usedThisFrame/solveModeUsed/arcsecPerPixel`)
- `guidance` (`arrowDx/arrowDy/remainArcsec/segmentRemainArcsec`)
- `statusText`

## 9. 日志关键字与排障路径
关键日志建议关注：
- `guidance tracking config`
- `tracking_fused`
- `recovering_multi`
- `reanchor_solve`
- `fresh solve reanchor lock target`

排障路径：
1. 先看 `match.inliers/inlierRatio/reprojErrPx` 是否长期劣化。
2. 再看 `lostFrames` 是否在 `recovering_multi` 快速增长。
3. 若频繁 `reanchor_solve`，检查 `RECOVER_MAX_FRAMES`、RANSAC 阈值与星点质量。
4. 若角秒级轨迹一直单向漂移，看 `precision.driftPxPerSec` 是否持续非零、`compensatedRemainArcsec` 是否明显小于 `guidance.remainArcsec`。
5. 若过早 done，看 `precision.dynamicDoneThresholdArcsec` 是否被 `measurementNoiseArcsec` 抬得过高，可下调 `QUARCS_POLAR_DYNAMIC_THRESHOLD_SIGMA` 或检查解算重投影误差。
6. 若前端显示异常，抓取 `PolarImageGuidanceDataV2` 原始 JSON 校验字段完整性。
